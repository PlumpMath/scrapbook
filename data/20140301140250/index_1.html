                            Vinod Kurup Mod Dan • a year ago

                            Dan,

                            Yes, that's exactly the issue. I submitted that bug report yesterday, not expecting that he'd be so quick to fix it. I also thought that he would only remove the pymacs.el file, not the 'py-load-pymacs' function that I was still using. My bad for recommending to use the master branch on such a fast-moving project. There are 2 possible workarounds.

                            1. Copy the py-load-pymacs function to init.el so it will be available. The function is on lines 18269-18307 at this link: http://bazaar.launchpad.net/~p...
                            2. Use the stable version of python-mode (6.0.12) and make this small change. Add a slash to the name of the virtualenv. (virtualenv-workon 'default/'). That bug is fixed on master, but is easy enough to work around.
                            Unfortunately, I'm not near a real computer (A computer that only has IE7 isn't a real computer :-) ), so I haven't been able to test these scenarios, but I will and will update the post soon.

                            Thanks for the feedback!

                                Dan Vinod Kurup • a year ago

                                I backed up to python-mode 6.0.12 and added a slash to (virtualenv-workon "default/") and that got through initialization in a terminal.

                                To get initialization to complete when I started from the dock, I added exec-path-from-shell to my-packages and just below installing the packages, added this:

                                ;; Copy PATH, MANPATH, WORKON_HOME, exec-path from terminal environment
                                (when (memq window-system '(mac ns))
                                (let
                                ((exec-path-from-shell-variables '("PATH" "MANPATH" "WORKON_HOME")))
                                (exec-path-from-shell-initialize)))

                                I probably only needed that because I use a non-standard WORKON_HOME.
