    Hi Marius,
    Nice blog! Thanks! :)
    I have been using etags( ctags for emacs)/ctags for quite a long time. I never used tags-query-replace; quite exited about it.

    Here are few tips from my side:

    If your code never grows then you are fine, but that may not be the case or should not be the case ;). You may/must want to update your TAGS table quite often to find the tags for new code added by people/you. Hence, installing cron to refresh/update TAGS might be wise thing to do. man crontab for more info or cron
    Here is my crontab file which updates my TAGS every 2 hours:

    [sachin@dev202]$ crontab -l
    0 */2 * * * cd /home/sachin/ROR/; ctags -e -a --Ruby-kinds=-f -o TAGS -R .

    Also you may want to add the following elisp to your .emacsfile:


    ;;by default the emacs will search the etags (TAGS) in pwd. Changing the default and pointing to your TAGS file
    ;; My tag file is @ /home/sachin/ROR/TAGS

    (let ((tags-table (concat (getenv "HOME") "/ROR/TAGS")))
    (when (file-exists-p tags-table)
    (visit-tags-table tags-table)))

    Cheers!!!!
    Sachin
