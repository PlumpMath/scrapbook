                I have found that you need this to make the completion mode work as described:
                (require 'anything-ipython)
                (define-key py-mode-map (kbd "M-") 'anything-ipython-complete)
                (define-key py-shell-map (kbd "M-") 'anything-ipython-complete)
                (define-key py-mode-map (kbd "C-c M") 'anything-ipython-import-modules-from-buffer)

                It was nestled in anything-ipython.el

                                                                                                    A well-written document. Thanks. I was just coming back to note the problem with anything-ipython completion not working, and I see that it's already discovered. There also seem to be some inconsistencies with the behavior of ipython invoked within emacs -- it doesn't behave the same as it does when invoked in a terminal. It does not seem to be possible to use it as a straight python interpreter. A trivial example:

                                                                                                    ipython in emacs:

                                                                                                    In [5]: l = "Name: this list"
                                                                                                    In [6]: l
                                                                                                    Out[6]: 'Name: this list'
                                                                                                    In [7]: v = l.split(':')
                                                                                                    In [8]: v
                                                                                                    Out[8]: ['Name', ' this list']
                                                                                                    In [9]: if v[0] == 'Name' :
                                                                                                    ...: print 'yes'
                                                                                                    Display all 297 possibilities? (y or n)

                                                                                                    ipython in xterm:
                                                                                                    In [1]: l = "Name: this list"
                                                                                                    In [2]: l
                                                                                                    Out[2]: 'Name: this list'
                                                                                                    In [3]: v = l.split(':')
                                                                                                    In [4]: v
                                                                                                    Out[4]: ['Name', ' this list']
                                                                                                    In [5]: if v[0] == 'Name' :
                                                                                                    ...: print 'yes'
                                                                                                    ...:
                                                                                                    yes

                                                                                                    In emacs, the shell will list all available commands available at the prompt, instead of just printing 'yes'. Without the ability to use the auto-completion option, I don't know that using ipython gives any benefit within emacs.

                                                                                                    Thanks again.
